1. A Description of the Github Workflow

Github is a website that holds repositories of code that developers can use to store their working files. It helps developers collaborate together on projects as Github repositories can be made accessible for teams working across locations and time zones.

Repositories on Github can consist of multiple files that make up a software project. These files can be accessed by multiple developers at any given time who all collaborate on the same project and contribute their own work to the repository.

A Github repository consists of ‘branches’, which are segments of code that developers use to collaborate together and work on their own changes to the code base. The ‘master branch’ is the central set of files in a repository which collates all the changes that have been approved and tested, and are ready to be deployed live as part of a website or application.

To make changes to the ‘master branch’, a separate branch must be created so changes can be implemented and tested. When a separate branch is created, it consists of a carbon copy of the ‘master branch’ at that specific moment in time. The code in this branch can then be changed and tested without affecting the ‘master branch’.

For changes to be implemented to the ‘master branch’, a developer must open a pull request. A pull request is the process of submitting and accepting changes made to the master branch to test these changes and ensure they work and meet the standards of the project. Once the pull request has been accepted, the changes made in the branch can be implemented to the master branch.

2. A Description of the Unix Shell

The Unix shell is a command line interface. Developers can use the Unix shell to manage their work files and automate processes that would otherwise be time consuming if carried out in an alternative graphical user interface. One of the most widely used Unix shells is Bash, which can be used on both Windows and Macintosh devices.

A Unix Shell can be operated using commands that consist of short abbreviated words. The table below details some of the most used commands for a Unix shell command line. Notice that some of the commands have single letter extensions (i.e ls -a). These are called flags and are appended to commands to carry out specific tasks in the command line.

Unix Commands
pwd Print the current working directory in the command line
ls List all files in a directory
ls -a List all files in a directory including hidden files
cd *file name* change directory to a particular file in the directory
cd .. return to parent file in the directory
mkdir *file name* create a new working directory
touch *file name* create a new file using the nano text editor
rm *file name* remove a file from the directory
